# Conference_-2
Модуль классификации ошибок времени выполнения с использованием методов NLP


Цель исследования: Автоматически categorize ошибки (Exception types) и предлагать возможные причины их возникновения, анализируя не только тип исключения, но и контекст его возникновения.
· Суть: Модуль анализирует стектрейсы, логи и состояние переменных в момент ошибки. Текстовые данные (имена функций, переменных, значений) обрабатываются с помощью техник NLP (например, преобразование в Bag of Words или эмбеддинги). Модель машинного обучения (например, Gradient Boosting или нейросеть) обучается относить ошибку к определенному классу (напр., «Ошибка связана с работой с базой данных», «Ошибка валидации входных данных»).
· Методы ИИ: Обработка естественного языка (NLP), классификационный ML.
· Выход: Интеллектуальный анализатор логов, который помогает разработчику быстрее понять категорию и источник ошибки.
